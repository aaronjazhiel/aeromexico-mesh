Funciones y Responsabilidades por Equipo

4 equipos · Cuentas AWS asignadas · Gobierno y modelo de colaboración

Equipos de plataforma

Equipo 01
Plataforma de Datos Mesh
Acelerar la entrega ágil de productos de datos para potenciar el crecimiento del negocio y la experiencia del cliente.
Producer Domain Account × 6 verticales · 1 cuenta AWS por vertical
Ciclo de vida del Data Product
  • Definir propósito, dueños y estándares de nombres
  • Solicitar cuenta AWS y configurar llaves KMS
  • Configurar Zona de Inspección · validar pipelines
  • Ejecutar ingesta batch y tiempo real (Kinesis, Kafka)
  • Construir Datasets, Modelos y Pipelines CI/CD
  • Revisar cifrado y Checklist de Seguridad
  • Despliegue con aprobación DevSecOps
  • Documentar ADR · registrar en Amazon DataZone
Gestión de la plataforma
  • Administrar SageMaker Unified Studio
  • Gestionar capas RAW → Curated → Purpose-Built
  • Mantener AWS Glue Data Catalog
  • Exponer datasets como Data As a Service
Herramientas clave
SageMaker Studio Amazon DataZone Glue / EMR S3 · Kinesis Athena · QuickSight Amazon Q Dev
Equipo 02
Plataforma Services Mesh APIs
Exponer, gobernar y controlar el consumo de APIs. Registro central de todo servicio expuesto en Sensedia.
API Layer dentro de cada Producer Account · Sensedia / Auth0 / Akamai = SaaS transversales
Administración del MESH API Services
  • Administrar API Products 1–4, Lambda JDBC y API GW
  • Gestionar autenticación M2M con Auth0 (SaaS)
  • Administrar Akamai CDN y WAF para exposición externa
  • Gestionar Transit Gateway para routing entre cuentas
Gobierno de APIs en Sensedia
  • Publicar y mantener contratos OAS/RAML
  • Registrar Data As a Service del equipo Data Mesh
  • Registrar APIs cognitivas del equipo AgentCore
  • Registrar endpoints de inferencia del equipo ML
  • Onboarding: Portal, Salesforce, GPTs · rate limits
Herramientas clave
Sensedia PaaS Auth0 M2M Akamai CDN/WAF Amazon API GW Lambda JDBC New Relic
Equipo 03
Plataforma IA Agentes / AgentCore
Operar agentes de IA que consumen datos del Data Mesh y exponen capacidades cognitivas via APIs en Services Mesh.
AMX AI Agentic Account + AMX AMAIA Account · MCP · Palo Alto FW
AgentCore — AMX AI Agentic Account
  • Administrar Facade, Gateway, Identity y Policies
  • Operar Token Manager y Token Cache MemoryDB
  • Gestionar SPAN Collector y Tool Registry
  • Configurar protocolo MCP
AMAIA y Runtime — AMX AMAIA Account
  • Operar AMAIA EKS + AMAIA-KB POD + PostgreSQL
  • Administrar AgentCore Runtime y Runtime API
  • Bedrock LLMs (Nova, Claude, GPT) + Guardrails
  • Definir reglas: AMAIA / Aerobot / Common
Herramientas clave
AgentCore Runtime Bedrock LLMs AMAIA EKS MemoryDB MCP · A2A Witness AI
Equipo 04
Plataforma Machine Learning
Construir, entrenar y desplegar modelos de ML que alimentan capacidades analíticas del Data Mesh y cognitivas de los agentes.
ML Account (propuesto) · SageMaker + Feature Store en la misma cuenta AWS
Pipeline de entrenamiento
  • Acceso a datasets curados via DataZone
  • Feature engineering en SageMaker + Feature Store
  • Diseñar experimentos y evaluar modelos (F1, AUC)
Registro, despliegue y MLOps
  • Registrar modelos en Model Registry con linaje
  • Desplegar endpoints con estrategia blue/green
  • Notificar a Services Mesh para registro en Sensedia
  • Notificar a AgentCore cuando hay nuevo modelo
  • Monitorear drift con CloudWatch · reentrenamiento
Herramientas clave
SageMaker Studio Feature Store Model Registry SM Pipelines SM Endpoints CloudWatch

Actores transversales

🛡

DevSecOps / Seguridad

Supervisa y aprueba transversalmente los 4 equipos

✓
Aprueba Checklist de seguridad de cada Data Product antes de habilitar la Zona de Inspección
✓
Valida IAM, KMS y Macie antes de cada despliegue a producción · Firma la aprobación DevSecOps
✓
Aprueba políticas OAuth / JWT de cada API registrada en Sensedia · Valida Auth0 M2M
✓
Supervisa Bedrock Guardrails y Witness AI en respuestas de agentes de IA
✓
Monitorea EXABEAM/EXPEL SIEM · Puede suspender accesos o aislar cuentas ante incidentes
📐

Arquitectura / Governance

Aprueba y gobierna el ciclo de vida de todos los activos

✓
Aprueba inicio de cada Data Product y valida que no exista duplicado en DataZone
✓
Valida ADR y publica en catálogo oficial · Notifica a ML y AgentCore cuando es relevante
✓
Aprueba publicación de APIs en Sensedia: versionado, no duplicidad, categorización correcta
✓
Aprueba caso de uso de modelos ML y valida Model Registry antes del despliegue a prod
✓
Gestiona ciclo de deprecación de APIs, Data Products y modelos obsoletos

Principio rector

⚡
Regla de oro
Todo servicio expuesto — datos, modelos ML o respuestas de agentes — se registra en Sensedia a través del equipo Services Mesh. Ningún equipo publica APIs directamente sin pasar por el proceso de registro y aprobación.
